# DB-Managed Profile Prompts

> Move profile prompts from hardcoded arrays to a database-managed system with 7 sections, 10 prompts each.

## Database Schema

### `prompt_sections`

| Column | Type | Notes |
|---|---|---|
| id | uuid | PK, default gen_random_uuid() |
| name | text | e.g., "Who I Am" |
| subtitle | text | e.g., "personality, quirks, daily life" |
| display_order | int | 1-7, Sweat Life = 7 |
| created_at | timestamptz | default now() |

### `prompts`

| Column | Type | Notes |
|---|---|---|
| id | uuid | PK, default gen_random_uuid() |
| section_id | uuid | FK → prompt_sections |
| prompt_text | text | e.g., "The thing most people don't realize about me until they know me well is..." |
| display_order | int | Ordering within section (1-10) |
| is_active | boolean | Default true, soft-disable without deleting |
| created_at | timestamptz | default now() |

### `profile_prompts`

| Column | Type | Notes |
|---|---|---|
| id | uuid | PK, default gen_random_uuid() |
| profile_id | uuid | FK → profiles |
| prompt_id | uuid | FK → prompts |
| section_id | uuid | FK → prompt_sections (denormalized for constraint) |
| answer | text | User's response (max 225 chars, enforced in app via existing APP.MAX_ONBOARDING_PROMPT_ANSWER_LENGTH) |
| engagement_count | int | Default 0, tracks chat icon taps |
| created_at | timestamptz | default now() |
| updated_at | timestamptz | default now() |

**Constraints:**
- `UNIQUE(profile_id, section_id)` — enforces one prompt per section per user

### Migration: Drop `approach_prompt`

- Drop the `approach_prompt` column from the `profiles` table (no longer needed).

### RLS Policies

- `prompt_sections` and `prompts`: read-only for authenticated users.
- `profile_prompts`: read for authenticated users (to view other profiles), insert/update/delete restricted to the owning user (`profile_id = auth.uid()`).

## Onboarding Flow

Replaces the current single prompts step with **7 individual screens** (one per section).

### Per-section screen:
1. Shows section name and subtitle as header
2. Lists the 10 prompts for that section
3. User taps a prompt to select it
4. Textarea appears for their answer
5. "Next" proceeds to the next section (disabled until prompt is selected and answer is written)

### Data flow:
- Fetch all sections + prompts from DB on entering the first prompt screen (single query, cached in onboarding store)
- Answers stored in onboarding store as the user progresses
- All 7 `profile_prompts` rows inserted when onboarding completes (in `complete.tsx`)

### Progress indicator:
- Onboarding step count increases from 9 to ~13 (replacing 1 prompts step with 7 section steps)
- Update progress bar/step counter accordingly

## Edit Profile

1. Profile edit screen shows all 7 prompt answers grouped by section
2. Tapping a prompt answer opens a section-specific bottom sheet with the 10 prompts
3. User can swap to a different prompt and/or edit their answer
4. Saving updates the `profile_prompts` row
5. `engagement_count` resets to 0 if the user swaps to a different prompt (new prompt/answer combo), stays unchanged if they only edit the answer text

## Profile Display (First Pass)

### Discover / SwipeDeck:
- Fetch `profile_prompts` joined with `prompts` and `prompt_sections` for each profile
- Ordered by `prompt_sections.display_order` (Sweat Life last)
- Replaces `mockPrompts` in `SwipeDeck/index.tsx`

### ProfileDetailSheet:
- Renders all 7 prompts in section display order using existing `PromptItem` component
- Chat icon on each prompt — tapping increments `engagement_count` and opens messaging

### Future enhancement (not in scope):
- Engagement-ranked layout: most-tapped prompt displayed as "hero" above bio, then bio, then 3 prompts, more bio, then 3 more prompts.

## API Layer (`lib/api/profiles.ts`)

- `fetchPromptSections()` — all sections with their prompts, ordered by display_order
- `fetchProfilePrompts(profileId)` — a profile's 7 answers with prompt text and section info
- `upsertProfilePrompt(profileId, sectionId, promptId, answer)` — save/update a prompt answer (used by onboarding and edit profile)
- `incrementEngagement(profilePromptId)` — increment engagement_count
- Remove references to `approach_prompt`
